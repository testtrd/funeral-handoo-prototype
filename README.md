# 業務引継書プロトタイプ

葬儀搬送時の「業務引継書」をタブレットで入力・確認・保存するための Next.js / React / TypeScript プロトタイプです。

## 実装済み

- 拠点選択、本社拠点
- 本社 8 社の業者マスター
- 業者別ルールによる表示制御
- 家族葬のファミーユのみ喪主・代表者生年月日を表示・必須化
- 家族葬のファミーユ、花てまりのみ火葬予約状況「済」を最終送信条件にする制御
- 家族葬のファミーユ、ジェイエイやすらぎセンターのみ外部問い合わせ回答を表示
- 全業者共通のペースメーカー有無の必須入力
- お迎え時間の入力
- 元号入力 UI と内部 ISO 形式保存
- 喪主・代表者住所の入力と帳票表示
- 故人住所の「喪主・代表者と同じ / その他」切り替えと、PDF向け「同上」表示
- 親族入力完了後のドライバー返却案内
- ドライバー追加入力画面
- 死亡診断書、死体検案書、処置、死亡日時をドライバー入力へ移動
- 死亡日時の通常・推定・頃・不詳・その他入力と displayText 保存
- 故人生年月日と死亡日からの満年齢自動計算
- 喪主・代表者の氏名フリガナ表記を「氏名（フリガナ）」へ変更
- 連絡希望先を「自宅 / 携帯 / 上記以外」へ変更し、上記以外のみ追加入力を表示
- お迎え先・搬送先、電話連絡日時、葬儀社連絡先、遺影写真を親族入力前のドライバー初期入力へ移動
- 故人基本情報から安置情報と満年齢表示を削除
- 宗教者関連から宗教者連絡先を削除し、紹介希望時のみ「希望宗派」表記へ変更
- 葬儀日程・火葬予約を元号なしの月日・時間入力へ変更
- 枕経等日時に「済み / 未定 / 無し / 頃」の状態を追加
- 使用品・備品をグループ化し、枕飾りセットと面布を選択式へ変更
- 添付の紙帳票に近い枠線付き印刷レイアウトへ調整
- 各画面上部に選択中の業者名を表示
- 電話連絡を日付なしの「電話連絡時間」へ変更
- 遺影写真を業者別追加質問へ移動
- 喪主・代表者、故人住所、宗教者、処置、同意の選択UIをコンパクト化
- 宗教者との関係を「菩提寺等 / 新家 / 不明」へ変更
- 火葬欄に待合室、予約番号を追加し、出棺時間の下に火葬時間を配置
- 受付日・受付時間・終了時間・確認印を入力フローから削除
- 内容確認画面のタイトルを「業務引継書（業者名）」へ変更
- 故人生年月日の元号に大正を追加
- 業者別追加質問画面を共通確認と業者別質問に分離
- 通夜・葬儀に希望欄を追加
- 使用品に樒欄を追加
- ドライアイスを「10kg / 20kg / 使用なし」の3択へ変更
- 枕飾りセット、面布、棺、木魚セット貸出Noに使用なし等の選択肢を追加
- 棺、木魚セット貸出Noは候補選択と手入力の両方に対応
- 電話連絡を月・日・時間の入力へ変更
- 遺影写真の選択肢を「有 / 未定 / 無」に整理
- 通夜・葬儀の希望欄から「不要 / 未確認」を削除
- 火葬欄は予約番号を待合室より先に表示
- 樒を「1束 / 2束 / 使用なし」に変更
- 棺を「使用なし / 手入力」に変更し、手入力時のみ内容欄を表示
- 面布を「使用あり / 使用なし」に変更
- 火葬予約前確認画面
- 帳票風の最終確認画面
- ブラウザ印刷による PDF 作成
- localStorage による途中保存・復元
- JSON エクスポート / インポート

## 起動

この環境では通常の `node` が PATH に無い場合があります。Codex 同梱 Node を使う場合は次のコマンドで起動できます。

```powershell
.\start-dev.cmd
```

PowerShell スクリプトで起動する場合は、実行ポリシーにより直接実行できないことがあります。その場合は次の形で起動してください。

```powershell
powershell -ExecutionPolicy Bypass -File .\start-dev.ps1
```

または直接起動する場合は次のコマンドを使ってください。

```powershell
& 'C:\Users\user\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' '.\node_modules\next\dist\bin\next' dev -p 3000
```

起動後に http://localhost:3000 を開いてください。
